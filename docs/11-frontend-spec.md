# 11 — 前端規格（UI / UX）

## 11.1 設計靈感與核心理念

本平台前端的核心體驗參考**鋼鐵雄心 4（Hearts of Iron IV）** 的世界地圖介面：

- 地圖是**主角**，不是附件——佔滿主要視窗區域
- 底部固定**時間軸**，可播放歷史、暫停、拖曳至特定日期
- 地圖色彩依緊張度**即時染色**，隨時間軸變化而動態更新
- 點擊國家 / 區域 → 浮現側邊資訊面板，不離開地圖頁面
- 整體配色走**暗色調**（深藍 / 深灰為底），突顯緊張度的紅橙漸層

---

## 11.2 技術選型

| 需求 | 選擇 | 理由 |
|---|---|---|
| 世界地圖底圖 | **Mapbox GL JS** | WebGL 渲染，支援流暢縮放、自訂樣式、熱力層；也可選 Leaflet（較輕量） |
| 國家填色層 | Mapbox GeoJSON Layer | 依 `net_tension` 動態設定 `fill-color` |
| 圖表（趨勢線、維度雷達） | **ECharts** | 時序折線、雷達圖、長條圖原生支援優秀 |
| 時間軸元件 | 自製 React 元件 | 需高度客製化，參考 HoI4 樣式 |
| UI 框架 | **Tailwind CSS** | Utility-first，易於打造暗色調客製介面 |
| 前端框架 | **React 18** | 生態成熟，狀態管理（Zustand）整合方便 |
| 狀態管理 | **Zustand** | 輕量，適合地圖 + 時間軸的全局狀態同步 |
| 動畫 | **Framer Motion** | 面板滑入、分數數字動畫 |

## 11.5 世界地圖（核心模組）

### 11.5.1 地圖圖層設計

```
Layer 順序（由底到頂）：
1. 底圖（Mapbox Dark Style / 自訂暗色底圖）
2. 國家填色層（country-fill）     ← 依 net_tension 染色
3. 國家邊界層（country-border）   ← 細白線，hover 時加亮
4. 事件熱點層（event-markers）    ← 圓點標記重大事件
5. 敏感地區標示層（hotspot-labels）← 台海、霍爾木茲等固定標示
6. Hover / 選取高亮層
```

### 11.5.2 國家填色色階

依 `net_tension` 值映射顏色（暗色調設計）：

| 分數範圍 | 顏色 | 說明 |
|---|---|---|
| 0–19 | `#1a3a2a`（暗綠） | 平穩 |
| 20–39 | `#2d3a1a`（黃綠） | 關注 |
| 40–59 | `#4a3a0a`（暗黃） | 升溫 |
| 60–79 | `#5a2a0a`（橙棕） | 高壓 |
| 80–100 | `#6a0a0a`（深紅） | 危機 |
| 無資料 | `#1a1a2a`（暗灰藍） | N/A |

> **注意**：評分引擎使用 `tanh` 壓縮正規化（見 `03-scoring-engine §3.4`），高分區間（80–100）在現實中較難達到。色階中段（40–60）在一般局勢下為最常用範圍，設計時可加強此段的色彩對比度以提升可讀性。

Mapbox expression 實作：
```json
[
  "interpolate", ["linear"],
  ["get", "net_tension"],
  0,   "#1a3a2a",
  20,  "#2d3a1a",
  40,  "#4a3a0a",
  60,  "#5a2a0a",
  80,  "#6a0a0a",
  100, "#8a0000"
]
```

### 11.5.3 Hover 互動

滑鼠移到國家上：
- 該國邊界高亮（白色 2px）
- 顯示 Tooltip：
  ```
  ┌─────────────────────┐
  │  🇮🇷 伊朗             │
  │  緊張度：87.3  危機   │
  │  ↑5.2 較昨日         │
  │  主因：飛彈攻擊事件   │
  └─────────────────────┘
  ```

### 11.5.4 點擊互動 → 側邊面板

點擊國家後，右側滑入**國家詳情面板**（見 11.7）。
點擊空白處關閉面板，地圖恢復正常狀態。

### 11.5.5 事件標記點（Event Markers）

重大事件（`final_score > 10`）在地圖上以**脈衝圓點**標示：

- **risk 事件**：紅色脈衝圓點（CSS pulse animation）
- **relief 事件**：綠色靜態圓點
- 點擊圓點 → 顯示事件摘要 Popup
- 數量太多時自動 cluster（聚合成數字氣泡）

### 11.5.6 敏感地區固定標示

以下地區始終顯示半透明標籤（不隨時間變化）：

```
台灣海峽  /  South China Sea  /  Strait of Hormuz
Red Sea  /  Korean Peninsula  /  Ukraine Border
```

## 11.6 底部時間軸（Timeline — 核心功能）

### 11.6.1 外觀設計（HoI4 風格）

```
┌──────────────────────────────────────────────────────────────────┐
│  ◀◀  ▶  ▶▶    ━━━━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  [最舊]                   2026-04-07                       [今日] │
│                                                                  │
│  速度：[慢] [●中] [快]           ● 有事件  ● 重大事件            │
└──────────────────────────────────────────────────────────────────┘
```

### 11.6.2 互動行為

| 操作 | 行為 |
|---|---|
| 拖曳滑桿 | 地圖即時更新為該日期的緊張度染色 |
| 點擊播放 ▶ | 自動前進，每步跳一天，地圖動態更新 |
| 點擊 ▶▶ | 加速播放（跳 7 天/步）|
| 點擊 ◀◀ | 倒帶（跳 7 天/步）|
| 暫停 ⏸ | 停在當前日期 |
| 點擊刻度上的事件標記 | 在時間軸上標示的重大事件點，點擊後跳至該日期 |
| 鍵盤 Space | 播放/暫停 |
| 鍵盤 ← / → | 前後一天 |

### 11.6.3 時間軸刻度標記

- 每個月顯示月份標籤
- 重大事件日期以**小三角形**標記在刻度上（`final_score > 15` 的事件）
- 顏色：risk 事件用紅色三角、relief 事件用綠色三角

### 11.6.4 動畫過渡

時間軸切換日期時，地圖顏色以 **400ms ease-in-out 漸變**更新（Mapbox transition），避免閃爍。

### 11.6.5 資料載入策略

```
預載策略：
- 載入當前日期 ±30 天的資料快取至前端記憶體
- 播放時若接近邊界，背景預載下一批 30 天資料
- 超過 90 天前的歷史資料按需載入（on-demand）

API 設計配合：
GET /api/map/heat?date=2026-04-07&dimension=overall
→ 回傳單日所有國家緊張度（用於地圖染色）

批次取得：
GET /api/map/heat/range?from=2026-03-01&to=2026-04-07&dimension=overall
→ 回傳日期範圍的完整資料（時間軸播放用）
```

## 11.13 全局狀態設計（Zustand）

```typescript
interface GlobalStore {
  // 時間軸狀態
  currentDate: string          // 'YYYY-MM-DD'
  isPlaying: boolean
  playSpeed: 'slow' | 'normal' | 'fast'  // 1天/s | 3天/s | 7天/s
  dateRange: { from: string; to: string }

  // 地圖狀態
  mapDimension: 'overall' | 'military' | 'political' | 'economic' | 'social' | 'cyber'
  selectedCountry: string | null   // ISO alpha-3
  selectedRegion: string | null    // 區域代碼（如 'middle_east'），區域頁面或區域摘要面板用
  mapData: Record<string, CountryHeatData>  // 快取當前日期地圖資料
  isMapLoading: boolean            // 地圖資料載入中（切換日期時顯示 loading 遮罩）
  preloadedRange: { from: string; to: string } | null  // 已預載至記憶體的日期範圍

  // UI 狀態
  sidePanelOpen: boolean
  sidePanelCountry: string | null
  error: string | null             // 全局錯誤訊息（API 失敗、網路斷線等），顯示 toast 通知

  // Actions
  setDate: (date: string) => void
  play: () => void
  pause: () => void
  setDimension: (dim: string) => void
  selectCountry: (code: string | null) => void
  selectRegion: (code: string | null) => void
  setMapLoading: (loading: boolean) => void
  setPreloadedRange: (range: { from: string; to: string } | null) => void
  setError: (msg: string | null) => void
}
```

## 11.14 新增 API 需求（配合時間軸）

前端時間軸功能需要後端新增以下 API（補充至 `05-api-spec.md`）：

### `GET /api/map/heat/range`

批次取得日期範圍的地圖資料，供時間軸播放預載使用。

```
Query Params:
  from=2026-03-01
  to=2026-04-07
  dimension=overall

Response:
{
  "dimension": "overall",
  "dates": {
    "2026-03-01": {
      "IRN": { "score": 82.1, "band": "Crisis", "band_zh": "危機" },
      "USA": { "score": 35.2, "band": "Watch",  "band_zh": "關注" }
    },
    "2026-03-02": { ... },
    ...
  }
}
```

### `GET /api/events-timeline`

取得日期範圍內的重大事件（用於時間軸刻度標記）。

> 注意：使用 `/api/events-timeline`（含 hyphen），避免與 `/api/events/{event_id}` 路由衝突（見 `05-api-spec §5.9`）。

```
Query Params:
  from=2026-03-01
  to=2026-04-07
  min_score=10          // 只取重大事件

Response:
{
  "events": [
    {
      "event_id": "evt_20260407_001",
      "event_time": "2026-04-07T08:00:00Z",
      "date": "2026-04-07",
      "title": "...",
      "risk_or_relief": "risk",
      "final_score": 18.5,
      "countries": ["IRN", "ISR"]
    }
  ]
}
```

## 11.15 效能優化策略

| 策略 | 說明 |
|---|---|
| 地圖資料預載 | 播放時預載前後 30 天資料至記憶體 |
| Mapbox 資料更新 | 只更新 GeoJSON source，不重新建立 Layer |
| React 渲染優化 | 地圖元件用 `React.memo`，避免父元件重渲染觸發地圖重繪 |
| 時間軸 debounce | 拖曳時 debounce 200ms 再觸發 API 請求（避免高頻觸發）|
| 事件列表虛擬捲動 | 使用 `react-virtual` 處理大量事件清單 |
| 圖片/字體預載 | 旗幟 emoji / 地圖 tile 預快取 |

## 11.16 響應式設計

| 裝置 | 調整策略 |
|---|---|
| 桌機（1440px+） | 完整版：地圖全螢幕 + 側邊面板 + 底部時間軸 |
| 筆電（1024px） | 同桌機，側邊面板縮小 |
| 平板（768px） | 側邊面板改為底部 drawer；時間軸縮簡版 |
| 手機（< 768px） | 隱藏地圖時間軸播放功能；改為 Dashboard 卡片版面 |
